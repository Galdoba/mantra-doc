**Кодек (от английского co-der/dec-oder — Кодировщик/Декодировщик)** — это комплексный **алгоритмический стандарт (спецификация)**, описывающий математические методы и шаги для **сжатия (уменьшения объема) цифровых аудио- и видеоданных при кодировании (энкодинге) и их последующего восстановления при декодировании**.

### Ключевые аспекты:

1.  **Это спецификация.** На бытовом уровне «кодеком» называют небольшую программу-декодер. Но технически это, в первую очередь, **стандарт** (как, например, рецепт). Реализации этого стандарта в виде библиотек (libx264, libvpx) или аппаратных схем — это уже **энкодеры** и **декодеры**.
    *   **Энкодер (кодировщик):** Программа, которая по стандарту кодека сжимает («конвертирует») исходные данные.
    *   **Декодер:** Программа, которая читает сжатые данные и восстанавливает их в форму, пригодную для воспроизведения.

2.  **Цель: компромисс между размером, качеством и сложностью.** Кодек существует для решения трилеммы:
    *   **Минимизировать объем данных** (битрейт).
    *   **Максимизировать качество** (сохранение деталей изображения/звука).
    *   **Минимизировать вычислительную сложность** для кодирования и, особенно, для декодирования (важно для стриминга и мобильных устройств).

3.  **Принципы сжатия (как это работает):**
    *   **Сжатие без потерь (Lossless):** Алгоритм находит и устраняет статистическую избыточность. Качество сохраняется идеально, но степень сжатия для медиа невелика (пример: FLAC для аудио, FFV1 для видео в архивах).
    *   **Сжатие с потерями (Lossy — используется в 99% случаев):** Алгоритм **безвозвратно отбрасывает данные**, которые считаются наименее важными для человеческого восприятия (психоакустическая и психо-визуальная модели).
        *   **Для видео:** Удаление пространственной избыточности (похожие участки в кадре) и временнóй избыточности (сходные области между соседними кадрами). Ключевые кадры (I‑кадры) сжимаются независимо, а предсказанные (P, B‑кадры) хранят лишь разницу.
        *   **Для аудио:** Удаление звуков за маской более громких тонов, сжатие динамического диапазона, упрощение спектра.

«Параметры кодирования» — это творческий процесс. Сам стандарт кодека (например, H.264) — это лишь набор допустимых инструментов. Поэтому один и тот же кодек может давать кардинально разный результат в разных руках.

Критически важное различие: 
- **Кодек** — это «что внутри», алгоритм сжатия. 
- **Контейнер (MP4, MKV, AVI)** — это «коробка», в которую упакованы потоки, сжатые этим кодеком. 
Один контейнер (например, MP4) может нести внутри потоки, сжатые разными кодеками.

### Таблица популярных кодеков

| Тип       | Кодек            | Основное применение                                     | Особенность                                                                        |
| :-------- | :--------------- | :------------------------------------------------------ | :--------------------------------------------------------------------------------- |
| **Видео** | **H.264 / AVC**  | Веб-видео (YouTube, стриминг), Blu-ray, камеры          | Отличный баланс качества/сжатия/совместимости. «Рабочая лошадка».                  |
|           | **H.265 / HEVC** | 4K/8K HDR-видео, современные стриминговые сервисы       | На ~50% эффективнее H.264 при том же качестве, но сложнее для декодирования.       |
|           | **AV1**          | Стриминг (YouTube, Netflix), видеоконференции           | Открытый, бесплатный. Эффективнее HEVC, но требует много ресурсов для кодирования. |
|           | **VP9**          | Предшественник AV1, YouTube, WebM                       | Открытый конкурент H.265.                                                          |
| **Аудио** | **AAC**          | Стандарт для видео (MP4, MKV), музыка (iTunes, Spotify) | Преемник MP3, лучшее качество при том же битрейте.                                 |
|           | **MP3**          | Устаревший, но повсеместно поддерживаемый формат музыки | Исторический стандарт, сейчас используется реже из-за низкой эффективности.        |
|           | **Opus**         | Веб-аудио (WebRTC, Discord), потоковое вещание          | Универсальный (от низких до высоких битрейтов), очень эффективный, открытый.       |
|           | **FLAC**         | Архивное хранение музыки, аудиофильские записи          | **Без потерь (Lossless)**. Полное сохранение качества, сжатие ~30-50%.             |
