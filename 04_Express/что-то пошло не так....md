Мы хотим создать единый скрипт, который будет обрабатывать видео с произвольным количеством аудиодорожек.  
Для этого мы будем использовать внешнюю утилиту `defineAudioStreams`, которая возвращает суффиксы для аудиодорожек.

План:

1. Определить количество и тип аудиодорожек, используя утилиту `defineAudioStreams`.
    
2. Динамически построить фильтр для аудио и видео, а также списки выходных файлов и параметров для `fflite`.
    

Предположим, что утилита `defineAudioStreams` принимает путь к файлу и возвращает список суффиксов, разделенных пробелами.

Мы можем получить этот список, запустив утилиту для исходного файла.

Однако, в текущих скриптах мы перемещаем исходный файл в директорию `PROGRESS_DIR`. Поэтому, чтобы проанализировать файл, мы должны сделать это до перемещения, либо после перемещения использовать путь в `PROGRESS_DIR`.

Давайте изменим порядок:

1. Определим аудиодорожки до перемещения файла (или после, но тогда используем путь в `PROGRESS_DIR`).
    
2. Построим фильтр и списки выходных файлов на основе этого списка.
    

Также, поскольку количество дорожек может быть любым, мы должны динамически генерировать:

- Аудио фильтры для каждой дорожки (с преобразованием и разделением на HD и proxy).
    
- Выходные файлы для каждой дорожки (HD и proxy).
    

В текущем скрипте (Film 2 Audio) мы имеем две дорожки, и мы вручную прописывали фильтры и выходные файлы для каждой.

Теперь мы хотим обобщить.

Шаги в скрипте:

1. Получить список суффиксов аудиодорожек (через `defineAudioStreams`).
    
2. Подсчитать количество дорожек (можно по количеству слов в выводе `defineAudioStreams`).
    
3. Построить строку фильтра для аудио и видео.
    

Видео фильтр будет одинаковым для любого количества дорожек: разделение видео на HD и proxy.

Аудио фильтр: для каждой дорожки (индекс от 0 до N-1) мы должны:  
- взять аудиодорожку [0:a:i] (где i - индекс дорожки)  
- применить к ней фильтр `AUDIO_FILTER` (который resample и atempo)  
- разделить на две дорожки: HD и proxy (с помощью asplit=2)

4. Сгенерировать параметры для `fflite` для каждого выхода.
    

Также нужно сгенерировать имена выходных файлов для каждой аудиодорожки.

Предлагаю следующую структуру:

Аудио дорожки будут обозначаться индексами: 0, 1, 2, ... и соответствующими суффиксами.

В фильтре мы для каждой дорожки создадим два потока: `[a_i]` для HD и `[a_i_pr]` для proxy, где i - индекс дорожки.

Затем в параметрах `fflite` мы будем мапить эти потоки в выходные файлы.

Имена выходных файлов для аудио будут формироваться так:  
<OUTPUT_BASE><REVISION_SUFFIX>_<SUFFIX>.m4a для HD  
<OUTPUT_BASE><REVISION_SUFFIX>_<SUFFIX>_proxy.ac3 для proxy

где <SUFFIX> - суффикс, полученный из утилиты для данной дорожки.

5. Также, возможно, мы захотим передавать язык и количество каналов в суффиксе, но это уже забота `defineAudioStreams`.
    

Реализация:

Мы будем использовать массивы в bash для хранения суффиксов и построения частей фильтра и параметров командной строки.

Пример:

text

    audio_suffixes=($(defineAudioStreams "${INPUT_DIR}/${INPUT_FILE}"))

Затем количество дорожек:

text

    audio_track_count=${#audio_suffixes[@]}

Построение аудио фильтра:

text

    audio_filter=""
    for i in ${!audio_suffixes[@]}; do
        audio_filter="${audio_filter} [0:a:${i}]${AUDIO_FILTER}[a${i}_in]; [a${i}_in]asplit=2[a${i}][a${i}_pr];"
    done

Общий фильтр:

text

    VIDEO_FILTER="[0:v:0]split=2[vidHD][inProxy]; [inProxy]scale=iw/${VIDEO_SCALE_DIVISOR}:ih, setsar=(1/1)*${VIDEO_SAR_FACTOR}[vidHD_pr];"
    FILTER_COMPLEX="${VIDEO_FILTER} ${audio_filter}"

Но обратите внимание, что в конце audio_filter будет лишняя точка с запятой, но это не страшно.

6. Построение параметров для аудио выходов в fflite:
    
    text
    

audio_output_params=""
for i in ${!audio_suffixes[@]}; do
    suffix=${audio_suffixes[$i]}
    audio_output_params="${audio_output_params} -map [a${i}] -c:a ${AUDIO_HD_CODEC} -compression_level ${AUDIO_HD_COMPRESSION_LEVEL} -map_metadata -1 -map_chapters -1 \"${FULL_EDIT_PATH}/${OUTPUT_BASE}${REVISION_SUFFIX}_${suffix}.m4a\""
    audio_output_params="${audio_output_params} -map [a${i}_pr] -c:a ${AUDIO_PROXY_CODEC} -b:a ${AUDIO_PROXY_BITRATE} \"${FULL_EDIT_PATH}/${OUTPUT_BASE}${REVISION_SUFFIX}_${suffix}_proxy.ac3\""
done
