**Stellar** — это строковое представление описания звезды или кратной звёздной системы, используемое в данных о мире или системе в игре Traveller. Он содержит один или несколько кодов спектральной классификации звёзд, объединённых в единую строку. Параметр позволяет однозначно определить тип, подтип, класс светимости каждой звезды в системе, а также её роль в кратной системе (первичная, компаньон и т.д.).

### Формат и структура

В программной реализации (на языке Go) тип `Stellar` определён как псевдоним для строки:

```go
type Stellar string
```

Строка может содержать последовательность кодов звёзд (`StarCode`), записанных подряд (без разделителей) или разделённых двоеточиями. Нормализованное представление использует двоеточия для разделения компонентов, например:

```
G2 V
F5 V:M5 V
A0 V:K2 V:M8 V
```

#### Компоненты Stellar

- **StarCode** — код спектральной классификации отдельной звезды. Включает:
  - Спектральный тип (`O`, `B`, `A`, `F`, `G`, `K`, `M` для звёзд главной последовательности; `L`, `T`, `Y` для коричневых карликов).
  - Числовой подтип (от `0` до `9`).
  - Класс светимости (`Ia`, `Ib`, `II`, `III`, `IV`, `V`, `VI`).
  
  Примеры: `G2 V` (жёлтый карлик, как Солнце), `M5 V` (красный карлик), `L3` (коричневый карлик).

- **Специальные коды** — для экзотических объектов:
  - `BD` — коричневый карлик (brown dwarf)
  - `D` — белый карлик (white dwarf)
  - `BH` — чёрная дыра (black hole)
  - `NS` — нейтронная звезда (neutron star)
  - `PSR` — пульсар (pulsar)
  - `pNb` — планетарная туманность (planetary nebula)

- **StarDesignation** — обозначение компонента в кратной системе. Используется при генерации случайных обозначений (например, `Aa` — первичная звезда, `Ab` — её близкий компаньон, `Ba` — следующий компонент и т.д.). Полный список:
  - `Aa` (Primary)
  - `Ab` (PrimaryComp)
  - `Ba` (Close)
  - `Bb` (CloseComp)
  - `Ca` (Near)
  - `Cb` (NearComp)
  - `Da` (Far)
  - `Db` (FarComp)

### Использование в Traveller

Параметр `Stellar` необходим для:
- **Генерации звёздных систем** — случайное определение количества и типов звёзд.
- **Определения характеристик планет** — спектральный класс звезды влияет на зону обитаемости, температуру, возможные типы миров.
- **Валидации данных** — проверка корректности введённого описания системы.
- **Отображения информации** — в карточке мира или системы игрокам и рефери.

### Примеры

| Описание системы | Строка Stellar |
|------------------|----------------|
| Одиночная звезда, похожая на Солнце | `G2 V` |
| Двойная система: жёлтый и красный карлики | `G2 V:M5 V` |
| Тройная система с белым карликом | `F5 V:K0 V:D` |
| Коричневый карлик | `L5` |
| Чёрная дыра с компаньоном | `BH:M8 V` |

### Программная обработка

В коде предусмотрены функции для:
- Разбора строки на отдельные коды (`ExtractStars`, `Split`).
- Валидации (`Validate`).
- Получения кода первичной звезды (`PrimaryCode`).
- Генерации случайных обозначений для кратных систем (`RollDesignations`, `RollStellarDesignations`).

Параметр **Stellar** может быть импортирован в функцию-генератор звёздной системы в качестве входных данных. Это позволяет рефери или сценаристу точно задать состав системы, минуя случайную генерацию — например, для воссоздания реальной астрономической системы, реализации сюжетного допущения или обеспечения строго определённых условий для последующего формирования планет. Генератор, принимая готовый код `Stellar`, будет использовать его как основу, дополняя систему орбитальными параметрами и другими характеристиками, соответствующими переданным спектральным классам.
